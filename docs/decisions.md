# decisions

## 2026-06-18

- PJ全体管理用の個人repoとして `used-car-ai-automation` を作成する
- 画像分類の実験・学習・評価repoとして `used-car-image-classification` を分ける
- 分類モデルは、まずResNet50をベースラインとして扱う
- ResNet50で精度が伸びない場合、ResNet50特徴量 + DeepForestを比較候補として検証する

## 2026-06-19以降

- PoC結果はモデル選定の最終結論ではなく、検証方法の理解材料として扱う
- AI継続改善は、生データ取得〜教師データ作成までのフローと、モデル学習・継続改善フローに分けて整理する
- アノテーション補助AIは汎用LLM/VLMを使い、SageMakerに載せる前提にしない
- モデル学習・評価・登録・デプロイはSageMakerを第一候補にする
- 機能要件は、モデル利用の有無ではなく、利用者がやりたいことのカテゴリで整理する
- 機能カテゴリは確定案ではなく叩き台とし、実現方式検討後に統合・分割を見直す
- API Gatewayは1つにし、Lambdaは責務ごとに分ける
- V0の通知表示は、WebSocketではなくVercel polling + `history-lambda` で進める
- 画像キャプション生成は、画像そのものではなく画像解析結果という構造データを入力にする
- 詳細分類は、画像分類精度改善だけでなく、後続処理を軽く・安定させるための前段構造化処理として検討する
- メインチャットはPJ全体のオーケストレーション、サイドチャットはissue単位の実作業に使う
- サイドチャットで決めたことは必ずファイルに残す
