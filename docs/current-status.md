# PJ Current Status

このファイルを見れば、現時点のPJ状況、課題、方針、進行中の作業が分かる状態にする。

## PJの目的

中古車業務における画像処理・書類読み取り・アノテーション・モデル改善の流れを整備し、AIを継続的に改善できる状態を作る。

## 現在の主な論点

### 1. 要件・TOBEがまだ固まっていない

渡辺さん側でTOBEを文章化する宿題が残っている。  
CAWAL側としては、TOBEが出てきた時に議論できるよう、現行フロー・課題・技術選定の判断材料を整理している。

### 2. プロダクト実行基盤をどう作るか

機能要件のカテゴリ整理は完了。  
現在は、カテゴリごとに実現方式を検討し、全体設計に進むための実装単位を固めている。

### 3. MLOps基盤をどう作るか

生データ取得〜教師データ作成までのフローと、モデル学習・継続改善フローは整理済み。  
モデル学習・評価・登録・デプロイは、SageMakerを第一候補としている。

### 4. モデル自体の処理フローに改善の余地がある

現行フロー:

```text
分類（内装・外装・広告）
  -> 検知（ナンバー・バナー）
  -> セグメンテーション
  -> インペイント
```

改善案:

```text
詳細分類（その画像がどういう画像なのかを認識）
  -> ルールベースで後続処理を振り分ける
```

詳細分類はまだ採用決定ではない。  
まずは機能要件ごとのやりたいことを整理し、既存モデル処理が適切かどうかを判断する。

### 5. 個人の画像認識知見が不足している

モデル自体の処理フローについて具体的に議論するため、画像分類、検知、セグメンテーション、インペイントの基礎と事例を学習した。  
今後は、機能要件ごとのモデル選定で使う。
